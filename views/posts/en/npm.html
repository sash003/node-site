Introduction

npm and Yarn are the most popular package managers for Node.js. They allow us to install libraries, manage versions, and even publish our own packages. In this article, we will discuss their differences, how to install dependencies, and how to create your own package.

What are npm and Yarn

npm — the built-in package manager, installed with Node.js. Yarn — an alternative from Facebook, focused on speed and stability. Both work through package.json, which stores the project dependencies.

Creating package.json


npm init -y
# or
yarn init -y

Installing dependencies

npm


npm install express
npm install lodash --save
npm install nodemon --save-dev

Yarn


yarn add express
yarn add lodash
yarn add nodemon --dev

Difference in lock files

npm uses package-lock.json, Yarn uses yarn.lock. These files lock dependency versions so the project works the same for all developers.

Removing a package


npm uninstall express
yarn remove express

Global packages


npm install -g nodemon
yarn global add nodemon

Scripts

In package.json you can store commands: 

"scripts": {
  "start": "node app.js",
  "dev": "nodemon app.js"
}
Run: 

npm run dev
yarn dev

How to create your own package

1. Create a new project folder. 2. Run npm init -y. 3. Add code in index.js. 4. In package.json specify "main": "index.js". 5. Log in: npm login. 6. Publish: npm publish.

Example of a simple package


// index.js
function greet(name) {
  return `Hello, ${name}!`;
}

module.exports = { greet };
Using it in another project: 

const { greet } = require('my-first-package');
console.log(greet("World"));

Comparing npm and Yarn

Criteria npm Yarn
Installation Comes bundled with Node.js Installed separately
Lock file package-lock.json yarn.lock
Speed npm 7+ became much faster, but still slower Faster for large number of packages
Caching Available, but less efficient Powerful caching and offline mode
Workspaces support Yes (since npm v7) Yes, implemented earlier
Compatibility De facto standard, supported everywhere Compatible with npm packages
Entry barrier Minimal, available immediately after Node.js installation Slightly higher, requires a separate tool

Conclusion

npm and Yarn solve the same task — dependency management. npm — the standard, comes with Node.js. Yarn — an alternative, works faster in large projects. Use either, but don't mix them in one project. For full experience, try creating and publishing your own package.
