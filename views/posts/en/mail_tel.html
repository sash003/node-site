Node.js is often used not only for web applications but also for background tasks: for example, sending notifications to users. The most popular options are sending emails and messages via Telegram. Let's go through both methods in practice.

Sending Emails via Nodemailer

For e-mail in Node.js, the package nodemailer is most commonly used. It works through an SMTP server (for example, Gmail, Mailgun, SendGrid, etc.). 

npm install nodemailer
Example of a simple script: 

// mail.js
const nodemailer = require('nodemailer');

// create transporter (SMTP connection)
let transporter = nodemailer.createTransport({
  service: 'gmail', // or smtp.yandex.ru, smtp.mail.ru
  auth: {
    user: 'yourmail@gmail.com',
    pass: 'app-password' // use app password
  }
});

// sending email
async function sendMail() {
  let info = await transporter.sendMail({
    from: '"My App" ',
    to: 'user@example.com',
    subject: 'Hello!',
    text: 'This is a test email from Node.js',
    html: 'This is a test email from Node.js'
  });

  console.log('Email sent: %s', info.messageId);
}

sendMail().catch(console.error);

Sending Notifications via Telegram

For Telegram notifications, you need to create a bot in BotFather. It will give you a token. Then get your chat_id by sending any message to the bot and checking https://api.telegram.org/bot<TOKEN>/getUpdates 

npm install node-fetch
Example code: 

// telegram.js
const fetch = require('node-fetch');

const TOKEN = 'YOUR_BOT_TOKEN';
const CHAT_ID = '123456789';

async function sendTelegram(text) {
  const url = `https://api.telegram.org/bot${TOKEN}/sendMessage`;
  const params = {
    chat_id: CHAT_ID,
    text: text,
    parse_mode: 'HTML'
  };

  const res = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(params)
  });

  const data = await res.json();
  console.log('Telegram response:', data);
}

sendTelegram('Node.js reports: server started!');

Combining Both Methods

You can move the functions sendMail() and sendTelegram() into a separate module and call them at any time: on user registration, on server errors, or when a new order arrives.
